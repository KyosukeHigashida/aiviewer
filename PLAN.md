## 履歴
### 完了: 最初の実装イテレーション
- React + TypeScript + Vite の最小構成を用意した。
- `react-pdf` で単一 PDF を表示できるようにした。
- 左に PDF、右にスレッドペインの 2 ペイン UI を作成した。
- PDF 上で選択したテキストを起点にスレッドを作成できるようにした。
- スレッドとコメントを `localStorage` に保存、復元できるようにした。
- 保存済みスレッドを右ペイン一覧で確認できるようにした。
- `react-pdf` の worker 読み込みを修正し、ローカル開発環境で PDF 表示を成立させた。

### 完了: 再発見 UI 改善
- スレッド作成時に `pageNumber`、`selectedText`、`contextBefore`、`contextAfter` を保存できるようにした。
- 保存済みスレッドに対応する位置マーカーを PDF 上の右端ピンとして表示できるようにした。
- 右ペインのスレッド一覧から、対象ページへスクロールし、対応するピンとスレッドを強調できるようにした。
- PDF 上の位置マーカーから、対応するスレッドを右ペインで見つけ直せるようにした。
- 本文テキストそのものの再ハイライトを追加した。
- 位置情報のない旧データを再発見対象外として区別表示できるようにした。

### 完了: ローカル PDF 読込対応
- ファイル選択 UI からローカル PDF を開けるようにした。
- PDF ごとの識別子として SHA-256 指紋を作成するようにした。
- 注釈スレッドを文書指紋ごとに保存、表示するようにした。
- 現在表示中の PDF に紐づくスレッドだけを右ペインと PDF 側に表示するようにした。
- PDF 未選択時の空状態を用意した。
- 旧データは区別表示し、現在の PDF と自動で混ざらないようにした。

### 完了: 常時見えるスレッドペインと独立スクロール
- デスクトップでは、PDF ペインとスレッドペインを独立してスクロールできるようにした。
- デスクトップでは、右ペインの新規スレッド作成部を常に見える位置に保ち、保存済みスレッド一覧を右ペイン内部だけでスクロールできるようにした。
- モバイルでは、画面下に常時見える 2 段階ボトムシートとしてスレッドペインを配置した。
- モバイルでは、閉状態でもハンドルからシートを引き出せるようにし、展開時に保存済みスレッド一覧を確認できるようにした。
- 一覧クリックやピンクリックで PDF を移動しても、右ペインの表示位置を失わないようにした。

### 完了: ハイライト安定化とモバイルシート修正
- 新規スレッド保存時に、本文ハイライト用の span アンカー情報を保存できるようにした。
- 本文ハイライトは、保存済みの span アンカーを優先して再描画するようにした。
- 既存スレッドは従来の文字列探索をフォールバックとして維持した。
- モバイルの保存済みスレッド一覧が新規スレッド入力部に重ならないように、シート内部のレイアウトを修正した。

### 完了: 保存済み矩形による本文ハイライト安定化
- 新規スレッド保存時に、選択範囲の `DOMRange` から取得した矩形群を保存できるようにした。
- 本文ハイライトは、保存済み矩形を最優先で再描画するようにした。
- 既存スレッドは span アンカーや文字列探索をフォールバックとして維持した。

### 完了: PDF 本体と前回文書の再開
- `IndexedDB` に PDF 本体、注釈スレッド、前回文書の再開状態を保存できるようにした。
- 起動時に前回開いていた PDF と、その文書指紋に紐づく注釈スレッドを自動で復元するようにした。
- 既存の `localStorage` 注釈は初回読み込み時に移行し、旧データは失わないようにした。
